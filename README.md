# update-container-action

GitHub Action to automatically update container image digests in Tinfoil configuration files (`tinfoil-config.yml`) and create versioned release tags for Tinfoil Containers repos.

This image will:
- Update `tinfoil-config.yml` with the provided new container image digest and version
- Create and merge a PR with the changes
- Push a new version tag pointing to the updated base branch

## Usage

```yaml
# Build the container
container-build:
  steps:
    - name: ...

config-update-and-tag:
  needs: container-build
  runs-on: ubuntu-latest
  permissions:
    contents: write
    pull-requests: write

  steps:
    - name: Update tinfoil-config.yml and create tag
      uses: tinfoilsh/update-container-action@329555b64fe9685196b91e6598ca06b4647b8821 # v0.3.0
      with:
        image: "ghcr.io/tinfoilsh/my-container"
        version: ${{ inputs.version }}
        digest: ${{ needs.container-build.outputs.digest }}
        github-token: ${{ secrets.GITHUB_TOKEN }}
        base-branch: ${{ github.ref_name }}
    - ...
```

## Prerequisites

The action expects a `tinfoil-config.yml` file in the repository root containing an image reference in the format:

```yaml
image: "<repo>@sha256:..." # v0.0.6
```

This action will create a PR and write to the repository and so requires the following workflow permissions:

```yaml
permissions:
  contents: write
  pull-requests: write
```

This action attempts to force merge the autogenerated PR without waiting for requirements.
This requires that the triggering user have administrator privileges in the repository.
