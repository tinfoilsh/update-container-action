# update-container-action

GitHub Action for Tinfoil Containers repos. It runs up to two independent,
opt-in steps for a container release:

- **Update config** (`update-config`, default `false`) — pins the container
  image digest(s) in `tinfoil-config.yml` and merges the change via a PR.
- **Create tag** (`create-tag`, default `true`) — pushes a `version` tag
  pointing at the base branch.

Either step can be enabled on its own. Both are idempotent: re-running with the
same inputs does not create duplicate PRs or tags.

## Inputs

| Input | Required | Default | Description |
|---|---|---|---|
| `image-refs` | when `update-config` is `true` | — | Container image references in `<repo>@sha256:<digest>` format, one per line. |
| `version` | yes | — | Version tag (e.g. `v0.0.6`). Used for the tag name and the config comment. |
| `github-token` | yes | — | Token used for PR operations and pushing the tag. |
| `base-branch` | yes | — | Branch targeted by the PR and the version tag. |
| `update-config` | no | `false` | Update `tinfoil-config.yml` with the provided digests via a merged PR. |
| `create-tag` | no | `true` | Create and push the version tag. |

## Usage

Tag only (the default — no config changes):

```yaml
- name: Create release tag
  uses: tinfoilsh/update-container-action@329555b64fe9685196b91e6598ca06b4647b8821 # v0.3.0
  with:
    version: ${{ inputs.version }}
    github-token: ${{ secrets.GITHUB_TOKEN }}
    base-branch: ${{ github.ref_name }}
```

Update `tinfoil-config.yml` and tag:

```yaml
config-update-and-tag:
  needs: container-build
  runs-on: ubuntu-latest
  permissions:
    contents: write
    pull-requests: write

  steps:
    - name: Update tinfoil-config.yml and create tag
      uses: tinfoilsh/update-container-action@329555b64fe9685196b91e6598ca06b4647b8821 # v0.3.0
      with:
        update-config: true
        image-refs: "ghcr.io/tinfoilsh/my-container@${{ needs.container-build.outputs.digest }}"
        version: ${{ inputs.version }}
        github-token: ${{ secrets.GITHUB_TOKEN }}
        base-branch: ${{ github.ref_name }}
```

For multiple images, pass a newline-separated list of references:

```yaml
      with:
        update-config: true
        image-refs: |
          ghcr.io/tinfoilsh/c1@${{ needs.build1.outputs.digest }}
          ghcr.io/tinfoilsh/c2@${{ needs.build2.outputs.digest }}
        version: ${{ inputs.version }}
        github-token: ${{ secrets.GITHUB_TOKEN }}
        base-branch: ${{ github.ref_name }}
```

## Prerequisites

When `update-config` is `true`, the repository root must contain a
`tinfoil-config.yml` with an image reference in the format:

```yaml
image: "<repo>@sha256:..." # v0.0.6
```

The config update opens a PR and writes to the repository, so the workflow
requires:

```yaml
permissions:
  contents: write
  pull-requests: write
```

(`contents: write` alone is sufficient when only `create-tag` is used.)

This action force-merges the autogenerated PR without waiting for required
checks. This requires the triggering user to have administrator privileges in
the repository.
